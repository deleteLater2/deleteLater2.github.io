How RSA Works (In the Real World)

Claim

This infographic How RSA Encryption Works is a dangerous oversimplification of how RSA works because it leaves out OAEP padding, where the keys come from, and how to fix input size limitations with AES.

OAEP

The article An Overview of RSA and OAEP Padding mentions that "RSA is a deterministic encryption which produces the same ciphertext given the same plaintext and key. An adversary can launch a chosen plaintext attack against the RSA encryption ... This attack would lead the leakage of important information of the plaintext." OAEP creates a probabilistic algorithm from the deterministic RSA algorithm, which is more secure. Without OAEP, RSA can be broken fairly easily.

AES

PKCS #1: RSA Cryptography Specifications Version 2.0 explains how RSA works, including information the infographic missed. It shows why AES with RSA is the norm, because the input message has to be, "an octet string of length at most k-2-2hLen, where k is the length in octets of the modulus n and hLen is the length in octets of the hash function output for EME-OAEP." Its recommended to encrypt the plaintext with AES and then encrypt the AES key with RSA so that the input never exceeds the max.

A sender gets the public key of the recipient, encrypts the plaintext with a new AES key, which is then padded with OAEP and encrypted with the recipient's public key. The sender sends both the encrypted and padded AES key as well as the ciphertext to the recipient. The recipient decrypts the AES key with their private key, unpads it, and uses it to decrypt the cipher text.
